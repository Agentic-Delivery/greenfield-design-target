# Handoff — Expense tracker, main screen

The customer approved this design and it was saved into the project automatically,
with no manual step in between. It is the approved look for the expense tracker's
main screen.

## What was approved
The home screen of a personal expense tracker: a running monthly total as an
editorial hero figure, an "add an expense" area (amount field, category chips, one
primary button), and a "recent" list grouped by day. The empty state, the
disabled-button state, and the add→success interaction are all designed and present.
It inherits the committed **Marginalia** brand verbatim (no new brand tokens) and
ships with a working dark mode.

## Build target
This is a **greenfield** design. The build target is the self-contained mockup at
`current/index.html` in this folder, with the design system described in
`design-system.md`. Per-viewport reference images live under
`current/viewports/index.html/`, and the machine-readable build target is in
`DESIGN-MANIFEST.json` with worker-fidelity instructions in `DESIGN-HANDOFF.md`.
